# Quickstart — первые шаги CopterLink

Этот чек-лист поможет быстро запустить базовую миссию, собрать логи и подготовить площадку для работы с разными каналами связи.

## 1) Базовая конфигурация
- **Автопилот**: ArduCopter (ArduPilot) 4.4/4.5, целевой контроллер — любой совместимый с Mission Planner.
- **Аппаратные профили**: начните с контроллера + приёмник RC, без внешних нагрузок.
- **Модули связи по умолчанию**: USB/телеметрия 57600/115200 для первичной настройки.
- **Питание**: отдельное питание для модема/плат расширения, общая земля с полётным контроллером.

## 2) Предполётная подготовка
1. Обновите прошивку контроллера и сбросьте параметры к дефолту при смене модели.
2. Калибруйте аксели/магнитометр/компас, задайте рамку и направление моторов.
3. Включите логирование (`LOG_DISARMED=1` на этапе отладки) и настройте частоту логов по каналу связи.
4. Настройте failsafe: `FS_THR_ENABLE`, `FS_GCS_ENABLE`, `FS_BATT_VOLTAGE/MAH` — под конкретный канал.
5. В Mission Planner загрузите простой план: взлёт → одна точка → возврат → посадка; сохраните копию в `docs/missions/`.

## 3) Связь (минимальная конфигурация)
- **LTE/GSM (Pi Zero + USB-модем)**
  - Настройте PPP/NCM и проброс порта MAVLink (например, `mavproxy --out udpin:0.0.0.0:14550 --out udp:GROUND_IP:14550`).
  - Проверьте ток потребления модема, добавьте DC-DC с запасом 1–1.5 А.
- **Wi‑Fi (ESP32-Dev/ESP32-CAM)**
  - Режим STA с фиксированным IP; прошивка для мостика UART↔UDP, скорость 57600/115200.
  - Тестовый стенд: подключение к контроллеру по UART, проверка RTT через Mission Planner.
- **LoRa (UART-модули)**
  - Выберите частоту/мощность под регион, установите низкую частоту MAVLink-стримов (`SRx_EXT_STAT`, `SRx_RAW_SENS`).
  - Тест: удержание связи на расстоянии 100–200 м с логом потерь.

## 4) Логи и артефакты
- Сразу после тестов выгрузите `tlog` и `dataflash` и поместите в `logs/` с именем `YYYY-MM-DD_channel_scenario`.
- Добавьте в описание: ветка/commit, тип связи, параметры failsafe, кратко — ветер/температура.
- Скриншоты HUD/Status Mission Planner сохраняйте рядом (`.png`/`.jpg`).

## 5) Следующие шаги
- Для каждого канала связи заведите ветку `comm/<channel>` и добавляйте туда скрипты/конфиги.
- Опишите обмен пользовательского оборудования с контроллером (MAVLink сообщениями или отдельным UART/UDP) в `docs/`.
- В hardware-папке фиксируйте схемы подключения модемов, уровни логики и питание.

Следуйте этому списку, чтобы быстро выйти на первый стабильный полёт и иметь воспроизводимые логи для сравнения каналов связи.
