# Roadmap и варианты связей

## Этапы
1. **Базовая миссия**: поднять коптер на выбранном полётном контроллере, выполнить простой маршрут по точкам, собрать лог.
2. **Интеграция пользовательского оборудования**: согласование питания, протокола обмена (MAVLink/Serial/UDP), проверка ответов от контроллера.
3. **Каналы связи**: обкатка и сравнение устойчивости для LTE/GSM, Wi‑Fi, LoRa; настройка приоритетов и резервирования.
4. **Навигация с optic flow**: подключение датчика, проверка посадки/висения, влияние на миссию.
5. **Стабилизация процессов**: шаблоны конфигурации Mission Planner, чек-листы дебага, автоматизация сборки прошивок.

## Требования по каналам связи
- **LTE/GSM**: наличие модема с поддержкой PPP/NCM, стабильное питание, fallback на SMS/USSD при потере данных.
- **Wi‑Fi**: режимы STA/AP, настройка статических адресов для телеметрии, проверка задержек в помехах.
- **LoRa**: выбор частоты и мощности под регион, длинные интервалы телеметрии, контроль duty cycle.
- **Optic flow**: датчик с выдачей скорости/высоты, проверка освещения и текстуры поверхности.

## Аппаратные платформы
- **Arduino Nano**: задачи управления периферией, простые датчики, обмен по UART/I2C.
- **Raspberry Pi Zero**: маршрутизация трафика, мини-сервисы обмена, запись логов.
- **ESP32-CAM / ESP32-Dev**: потоковое видео, телеметрия по Wi‑Fi, запас по GPIO для сенсоров.
- **Raspberry Pi Pico**: лёгкие драйверы, сопроцессор для датчиков.

## Работа с ветками
- Для каждой связи — отдельная ветка (`comm/...`) с собственными настройками и скриптами.
- Правки навигации и датчиков — ветка `nav/...`.
- Интеграция с наземными инструментами (Mission Planner, GCS) — ветка `integration/...`.
- Мёрдж в `main` только после полётных тестов и описанных логов.

## Логи и проверки
- Собираем `dataflash`/`tlog` после каждого полёта, складываем в `logs/`.
- К каждому логу — краткое описание миссии: высоты, точки, ветер, используемый канал связи.
- Фиксируем версии прошивок/скриптов и дату сборки.
